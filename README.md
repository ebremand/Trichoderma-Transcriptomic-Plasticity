## Project Overview

This repository contains scripts and pipelines used for genome annotation and transcriptomic analyses of *Trichoderma atroviride* strains confronted with three different plant pathogens.

The main analyses include:
- Functional genome annotation to identify CAZymes, peptidases, secondary metabolite biosynthetic gene clusters, secreted proteins, Gene Ontology (GO) terms, and PFAM domains
- Differential gene expression analysis in response to pathogen confrontation using AskoR

This study aims to understand transcriptional reprogramming in *T. atroviride* during interspecific interactions, with a particular focus on pathogen-dependent responses (*Alternaria brassicicola*, *Rhizoctonia solani*, and *Globisporangium ultimum*).

## Genome Annotation

Functional annotation was performed on the *T. atroviride* N1508 genome, where gene prediction was previously carried out using Helixer (https://www.biorxiv.org/content/10.64898/2026.06.22.733667v1).

**Objective:** Assign functional information to predicted genes and proteins.

Several tools were used in online or standalone mode; therefore, scripts are not always available. For reproducibility, text files describe the workflows and parameters used.

- **eggNOG.txt**  
  Functional annotation using eggNOG-mapper.  
  Assigns gene functions and PFAM domains.  
  Input: protein sequences  
  Output: functional annotation tables

- **dbCAN3.txt**  
  Identification of carbohydrate-active enzymes (CAZymes) using dbCAN3.  
  Input: protein sequences  
  Output: CAZyme classification tables

- **MEROPS.sh**  
  Identification and classification of peptidases using the MEROPS database.  
  Input: protein sequences  
  Output: peptidase annotation tables

- **SignalP.sh**  
  Prediction of secreted proteins.  
  Input: protein sequences  
  Output: list of predicted secreted proteins

- **Antismash.txt**  
  Detection of secondary metabolite biosynthetic gene clusters using antiSMASH.  
  Input: genome assemblies  
  Output: cluster predictions and summary tables

- **STRING.txt**  
  Retrieval of Gene Ontology (GO) annotations from the STRING database.  
  Input: protein sequences  
  Output: GO term annotations

## Transcriptomic Analysis

The transcriptomic pipeline consists of three main steps:

1. Read alignment and quantification  
2. Differential gene expression analysis  
3. Co-expression network analysis (WGCNA)

Each step is implemented using dedicated scripts to process RNA-Seq data and generate interpretable outputs.

---

### 1. Read Alignment and Quantification

**Objective:** Quantify gene expression and explore global transcriptomic patterns.

- **Salmon.sh**  
  Aligns RNA-Seq reads to the reference transcriptome using Salmon.  
  Input: paired-end FASTQ files and reference transcriptome  
  Output: transcript abundance estimates (TPM and raw counts)

---

### 2. Differential Expression Analysis

**Objective:** Identify genes significantly regulated between conditions.

- **RNAseq_DE_AskoR.R**  
  Performs differential expression analysis using AskoR.  
  Input: count matrices generated by Salmon and sample metadata  
  Output: lists of differentially expressed genes with log fold changes and adjusted p-values
